Auto-generate llms.txt, catalog.json, agents.md and 10 more AI-discovery files so ChatGPT, Claude, Perplexity and Google AI can find and sell your products.

== Description ==

**Will ChatGPT recommend your store?** Most WooCommerce stores don't emit the files AI shopping agents look for. This plugin ships **13 of them** — generated locally on your server, refreshed daily, and served straight from your own domain. Install, activate, done in 30 seconds. **Free forever.**

Unlike generic content-walking llms.txt plugins, this one is **commerce-aware**: it knows your products, prices, stock, variations, images and categories — and never reports a $0 price for a variable product, so your items render as real AI shopping cards instead of dropping to prose.

= The 13 files emitted =

* `/llms.txt` — discovery links + a card-ready product table (name + price + image + link) + AI shopping-assistant instructions
* `/index.md` — human-readable index: product count, categories, how to query
* `/llms-full.txt` — full catalog dump, one section per product, for LLMs that fetch everything at once
* `/catalog.json` — machine-readable ACP-compatible product feed
* `/products.json` — Shopify-shape feed for crawlers that assume Shopify
* `/agents.md` — agents.md emerging-standard discovery manifest
* `/sitemap-ai.xml` — sitemap optimized for AI crawlers (one URL per product)
* `/feed/google-shopping.xml` — Google Merchant Center RSS 2.0 product feed
* `/.well-known/agent-card.json` — A2A agent card
* `/.well-known/mcp.json` — MCP descriptor pointing at your local catalog
* `/.well-known/ucp` — Universal Commerce Protocol business profile
* `robots.txt` AI-bot directives — toggle which crawlers (ChatGPT-User, Claude-User, PerplexityBot, …) may access your store
* `<head>` discovery links + JSON-LD WebSite schema injected into your storefront

= Why commerce-aware matters =

Generic llms.txt plugins walk your blog posts and pages. AI shopping agents need **products** — with a numeric price and an image per item — to build a shopping card. This plugin reads your live WooCommerce catalog (including variable-product price ranges as "from" prices) so your products are card-ready the moment an agent fetches a file.

= Refresh =

Files refresh **daily via WP-Cron** and on demand from the Files tab. If a low-traffic site hasn't refreshed in 48 hours, the plugin schedules a catch-up refresh on the next page load so your files never go badly stale.

= Privacy =

Files are generated by PHP on your own server and written to your own webroot. By default the plugin also syncs each generated file's contents to xpay.sh so you get version history and one-click rollback — toggle this off in **Settings → LLMs.txt → Privacy** at any time. See `== External services ==` below for the full disclosure.

== External services ==

This plugin connects to xpay.sh to provide version control and one-click rollback for your generated AI-discovery files. This integration is **on by default** and disclosed in a dismissible admin notice on activation; you can switch it off at any time in **Settings → LLMs.txt → Privacy**.

**Service:** xpay.sh LLMs.txt version-control API
**Base URL:** `https://8mf8prh9rg.execute-api.us-east-1.amazonaws.com` (filterable via the `lltxt_backend_base_url` filter or the `lltxt_backend_base_url` option)
**Privacy policy:** https://xpay.sh/privacy
**Terms of service:** https://xpay.sh/terms

**Endpoints called, what's sent, when, and why:**

* `POST /v1/llms-txt/snapshot` — After every refresh (daily WP-Cron + on-demand). Sends the rendered file body for each route plus the site slug (derived from your domain), home URL, plugin/WordPress/WooCommerce version strings, and a sha256 of a randomly-generated local API key as the `X-Xpay-Api-Key` header. Lets you see prior versions and roll back.
* `GET /v1/llms-txt/versions` — When you open the Version Control tab. Sends the slug and API key hash only. Lists prior versions for a route.
* `GET /v1/llms-txt/version/{id}` — When you preview or restore a prior version. Sends the slug and API key hash only.
* `POST /v1/llms-txt/recommend` — When you click "Get recommendation" in the Version Control tab. Sends the slug, route, and API key hash. Returns a merged best-version body.
* `POST /v1/llms-txt/pin` — When you pin or unpin a version. Sends the slug, version id, pinned state, and API key hash.
* `DELETE /v1/llms-txt/merchant` — When you click "Delete my data" in the Privacy tab, or on plugin uninstall (if phone-home was enabled). Sends the slug and API key hash; wipes all your snapshots from xpay.sh.

**What is NOT sent:** no order data, no customer data, no admin credentials, no analytics, no the raw API key. Your file bodies are the same bodies your own `/llms.txt` etc. serve publicly.

== Installation ==

1. Upload the plugin to `/wp-content/plugins/` (or install from the WordPress Plugins screen).
2. Activate the plugin through the **Plugins** menu in WordPress.
3. Go to **Settings → LLMs.txt** to preview the generated files, toggle AI bots, and choose which products to surface.
4. Visit `https://yourstore.com/llms.txt` to confirm it's live.

== Frequently Asked Questions ==

= Does this send my data anywhere? =

By default the plugin syncs each generated file's contents to xpay.sh so you get version history and one-click rollback. You can disable this in **Settings → LLMs.txt → Privacy**, and the dismissible admin notice on activation discloses it up front. See the **External services** section above for the full list of endpoints, what's sent, and when.

= My host has a read-only filesystem — will it still work? =

Yes. If a static file can't be written, the plugin renders the file live on request as a fallback. You'll see the write error in the Diagnostics tab.

= I already have a robots.txt file. Will this overwrite it? =

No. The AI-bot directives are appended to WordPress's dynamic robots.txt and never overwrite a rule you've already set. If you have a *physical* robots.txt in your webroot, the plugin warns you and shows the directives to merge manually.

= Does it work with caching plugins (WP Rocket, LiteSpeed, Cloudflare)? =

Yes. The files are static and cache-friendly. If you change settings, regenerate from the Files tab and purge your cache.

= Why is GPTBot off by default? =

GPTBot is OpenAI's model-training crawler, not a shopping agent. The shopping/answer crawlers (ChatGPT-User, OAI-SearchBot, Claude-User, PerplexityBot, GoogleOther) are on by default; GPTBot is left off so you decide whether to allow training use.
